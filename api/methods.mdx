## Core Bridge Methods

These methods are available on the main NativeBridge instance for managing communication and configuration.

### execute()

Execute a native method and return a promise with the result.

```typescript
execute<T = any>(
  method: string,
  params?: any,
  options?: CallOptions
): Promise<T>
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `method` | `string` | Yes | Module and method name in format "Module.method" |
| `params` | `any` | No | Parameters to pass to the native method |
| `options` | `CallOptions` | No | Call-specific configuration options |

#### Returns

`Promise<T>` - Promise that resolves with the method result or rejects with a BridgeError.

#### Examples

```javascript
// Basic method call
const result = await bridge.execute('Device.getInfo');

// With parameters
const photo = await bridge.execute('Camera.takePicture', {
  quality: 'high',
  format: 'jpeg'
});

// With options
const data = await bridge.execute('API.longOperation', params, {
  timeout: 30000,
  priority: 'high'
});
```

### on()

Register an event listener for events emitted from native code.

```typescript
on<T = any>(
  event: string,
  listener: (data: T) => void
): () => void
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `event` | `string` | Yes | Event name to listen for |
| `listener` | `function` | Yes | Callback function to handle the event |

#### Returns

`function` - Unsubscribe function to remove the event listener.

#### Examples

```javascript
// Listen for device events
const unsubscribe = bridge.on('battery.levelChanged', (data) => {
  console.log('Battery level:', data.level);
});

// Unsubscribe when done
unsubscribe();
```

### emit()

Emit an event to native code.

```typescript
emit(event: string, data?: any): void
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `event` | `string` | Yes | Event name to emit |
| `data` | `any` | No | Data to send with the event |

#### Examples

```javascript
// Emit user action
bridge.emit('user.action', {
  type: 'button_click',
  target: 'purchase'
});
```

### registerModule()

Register a JavaScript module that can be called from native code.

```typescript
registerModule(
  name: string,
  module: ModuleDefinition
): void
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | `string` | Yes | Unique module name |
| `module` | `ModuleDefinition` | Yes | Object containing module methods |

#### Examples

```javascript
bridge.registerModule('Analytics', {
  track: (event, properties) => {
    // Implementation
  }
});
```

### configure()

Update bridge configuration at runtime.

```typescript
configure(options: Partial<BridgeOptions>): void
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `options` | `Partial<BridgeOptions>` | Yes | Configuration options to update |

#### Examples

```javascript
bridge.configure({
  debug: true,
  timeout: 15000
});
```

### isReady()

Check if the bridge is ready for communication.

```typescript
isReady(): Promise<boolean>
```

#### Returns

`Promise<boolean>` - Promise that resolves to true if bridge is ready.

#### Examples

```javascript
if (await bridge.isReady()) {
  // Bridge is ready
}
```

## Device Module Methods

Standard device information and control methods.

### Device.getInfo()

Get comprehensive device information.

```typescript
DeviceModule.getInfo(): Promise<DeviceInfo>
```

#### Returns

```typescript
interface DeviceInfo {
  model: string;           // Device model (e.g., "iPhone 14", "Pixel 6")
  manufacturer: string;    // Device manufacturer
  systemName: string;      // OS name ("iOS", "Android")
  systemVersion: string;   // OS version (e.g., "16.0", "13")
  identifier: string;      // Unique device identifier
  name: string;           // Device name set by user
  brand?: string;         // Device brand (Android only)
}
```

#### Examples

```javascript
const info = await bridge.execute('Device.getInfo');
console.log(`${info.manufacturer} ${info.model} running ${info.systemName} ${info.systemVersion}`);
```

### Device.vibrate()

Trigger device vibration.

```typescript
DeviceModule.vibrate(options?: VibrateOptions): Promise<void>
```

#### Parameters

```typescript
interface VibrateOptions {
  duration?: number;       // Vibration duration in milliseconds (default: 500)
  intensity?: 'light' | 'medium' | 'heavy'; // Vibration intensity (iOS only)
}
```

#### Examples

```javascript
// Default vibration
await bridge.execute('Device.vibrate');

// Custom duration
await bridge.execute('Device.vibrate', { duration: 1000 });

// With intensity (iOS)
await bridge.execute('Device.vibrate', { 
  duration: 300, 
  intensity: 'heavy' 
});
```

### Device.getBatteryLevel()

Get current battery level and charging status.

```typescript
DeviceModule.getBatteryLevel(): Promise<BatteryInfo>
```

#### Returns

```typescript
interface BatteryInfo {
  level: number;          // Battery level percentage (0-100)
  isCharging: boolean;    // Whether device is currently charging
}
```

#### Examples

```javascript
const battery = await bridge.execute('Device.getBatteryLevel');
if (battery.level < 20 && !battery.isCharging) {
  showLowBatteryWarning();
}
```

```javascript
  quality: 'medium',
```

## Storage Module Methods

Local storage and data persistence.

### Storage.get()

Retrieve a value from local storage.

```typescript
StorageModule.get<T = any>(key: string): Promise<T | null>
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `key` | `string` | Yes | Storage key to retrieve |

#### Returns

`Promise<T | null>` - The stored value or null if not found.

#### Examples

```javascript
// Get user preferences
const preferences = await bridge.execute('Storage.get', { key: 'user_preferences' });

// Get with type safety
const userId: string = await bridge.execute('Storage.get', { key: 'user_id' });
```

### Storage.set()

Store a value in local storage.

```typescript
StorageModule.set<T = any>(key: string, value: T): Promise<void>
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `key` | `string` | Yes | Storage key |
| `value` | `T` | Yes | Value to store |

#### Examples

```javascript
// Store user preferences
await bridge.execute('Storage.set', {
  key: 'user_preferences',
  value: { theme: 'dark', notifications: true }
});

// Store simple value
await bridge.execute('Storage.set', {
  key: 'last_login',
  value: Date.now()
});
```

### Storage.remove()

```typescript
StorageModule.remove(key: string): Promise<void>
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|

#### Examples

### Storage.clear()

Clear all stored data.

```typescript
StorageModule.clear(): Promise<void>
```

### Storage.getAllKeys()

## Network Module Methods

Network connectivity and HTTP requests.

### Network.getConnectionInfo()

Get current network connection information.

```typescript
NetworkModule.getConnectionInfo(): Promise<ConnectionInfo>
```

#### Returns

```typescript
interface ConnectionInfo {
  isConnected: boolean;
  connectionType: 'wifi' | 'cellular' | 'ethernet' | 'none';
  isMetered: boolean;     // Whether connection is metered (affects data usage)
  strength?: number;      // Signal strength (0-100, if available)
}
```

#### Examples

```javascript
const connection = await bridge.execute('Network.getConnectionInfo');
if (!connection.isConnected) {
  showOfflineMessage();
} else if (connection.isMetered) {
  askUserBeforeDownloading();
}
```

### Network.request()

Make an HTTP request using native networking.

```typescript
NetworkModule.request(options: RequestOptions): Promise<Response>
```

#### Parameters

```typescript
interface RequestOptions {
  url: string;
  method?: 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH';
  headers?: Record<string, string>;
  body?: any;
  timeout?: number;
  followRedirects?: boolean;
  validateSSL?: boolean;
}
```

#### Returns

```typescript
interface Response {
  status: number;
  statusText: string;
  headers: Record<string, string>;
  data: any;
}
```

#### Examples

```javascript
// GET request
const response = await bridge.execute('Network.request', {
  url: 'https://api.example.com/users',
  method: 'GET',
  headers: {
    'Authorization': 'Bearer ' + token
  }
});

// POST request
const createResponse = await bridge.execute('Network.request', {
  url: 'https://api.example.com/users',
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: {
    name: 'John Doe',
    email: 'john@example.com'
  }
});
```

Authenticate user using biometrics.


#### Parameters

```typescript

  subtitle?: string;      // Dialog subtitle
  description?: string;   // Dialog description

  negativeTitle?: string; // Cancel button title
```

#### Returns

```typescript
}
```

## Error Codes

Common error codes returned by bridge methods:

| Code | Description |
|------|-------------|
| `MODULE_NOT_FOUND` | The specified module is not registered |
| `METHOD_NOT_FOUND` | The specified method does not exist |
| `INVALID_PARAMS` | Invalid or missing parameters |
| `PERMISSION_DENIED` | Required permission not granted |
| `FEATURE_UNAVAILABLE` | Feature not available on this platform |
| `TIMEOUT` | Method call timed out |
| `NETWORK_ERROR` | Network operation failed |
| `NATIVE_ERROR` | Error occurred in native code |
| `SERIALIZATION_ERROR` | Failed to serialize/deserialize data |
| `BRIDGE_NOT_READY` | Bridge is not initialized |
| `OPERATION_CANCELLED` | Operation was cancelled by user |

## Method Conventions

### Parameter Validation

All methods should validate their parameters and return appropriate error codes:

```javascript
// Example parameter validation
if (!params.key || typeof params.key !== 'string') {
  throw new BridgeError('INVALID_PARAMS', 'key must be a non-empty string');
}
```

### Async Operations

Long-running operations should:
- Use appropriate timeouts
- Support cancellation where possible
- Provide progress updates via events

### Platform Differences

When methods behave differently across platforms:
- Document the differences clearly
- Provide feature detection methods
- Offer platform-specific alternatives

### Error Handling

Consistent error handling across all methods:
- Use standard error codes
- Provide descriptive error messages
- Include relevant context in error details
